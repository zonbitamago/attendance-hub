# Feature Specification: Glassmorphism Redesign - Sky Theme Light Mode

**Feature Branch**: `001-glassmorphism-redesign`
**Created**: 2025-11-20
**Status**: Draft
**Input**: User description: "全6ページをSkyテーマ（ライトモード専用）にデザインリニューアル。ThemeToggleコンポーネントとダークモード機能を完全削除し、Skyカラーパレット（sky-50〜sky-600）を全ページに適用。すべてのナビゲーションリンクをアイコン付きボタンスタイルに統一。グラデーション背景（bg-gradient-to-br from-white via-sky-50 to-white）とホバーエフェクトを実装。"

## User Scenarios & Testing

### User Story 1 - 統一されたSkyテーマのビジュアル体験 (Priority: P1)

ユーザーがアプリケーションのすべてのページにアクセスすると、一貫したSkyテーマのデザインで迎えられます。各ページで同じカラーパレット、グラデーション背景、カードスタイルが適用され、プロフェッショナルで洗練された印象を与えます。

**Why this priority**: これは最も基本的なユーザー体験です。視覚的な一貫性がなければ、ユーザーは混乱し、アプリケーションに対する信頼感が損なわれます。すべての機能の基盤となるデザインシステムです。

**Independent Test**: 全6ページ（トップ、イベント一覧、管理画面トップ、グループ管理、イベント詳細、一括登録）を順番に開き、すべてのページでSkyカラーパレット（sky-50〜sky-600）とグラデーション背景が適用されていることを目視で確認できます。

**Acceptance Scenarios**:

1. **Given** ユーザーがトップページを開いている、**When** ページを読み込む、**Then** 背景にグラデーション（白→sky-50→白）が表示され、カードが白色（border: sky-100）で表示される
2. **Given** ユーザーがイベント一覧ページを開いている、**When** ページを読み込む、**Then** トップページと同じグラデーション背景とカードスタイルが適用されている
3. **Given** ユーザーが管理画面を開いている、**When** ページを読み込む、**Then** 他のページと一貫したSkyテーマのデザインが表示される
4. **Given** ユーザーが任意のページを開いている、**When** ページ内のすべてのカード要素を確認する、**Then** すべてのカードが白背景、sky-100のボーダー、影付きで統一されている

---

### User Story 2 - 直感的なボタンスタイルナビゲーション (Priority: P2)

ユーザーがページ間を移動する際、すべてのナビゲーションリンクがアイコン付きのボタンスタイルで表示されます。これにより、クリック可能な要素が明確になり、操作性が向上します。

**Why this priority**: ナビゲーションの使いやすさは、ユーザーが目的の機能にスムーズにアクセスできるかどうかを決定します。視覚的に明確なボタンは、特にモバイルユーザーにとって重要です。

**Independent Test**: すべてのページでナビゲーションリンク（戻るボタン、管理画面、一括登録、ガイドなど）をクリックし、各リンクがアイコン付きのボタンスタイル（白背景、sky-200ボーダー、ホバー時sky-50背景）で表示されることを確認できます。

**Acceptance Scenarios**:

1. **Given** ユーザーがイベント一覧ページを開いている、**When** ナビゲーションエリアを見る、**Then** 「一括出欠登録」「管理画面」「使い方ガイド」の3つのボタンがアイコン付きで横並びに表示される
2. **Given** ユーザーがナビゲーションボタンにマウスをホバーする、**When** マウスカーソルをボタンの上に置く、**Then** ボタンの背景色がsky-50に変わり、ボーダー色がsky-300に変わり、テキスト色がsky-700に変わる
3. **Given** ユーザーが管理画面にいる、**When** 「トップページに戻る」リンクを確認する、**Then** 左矢印アイコン付きのボタンスタイルで表示される
4. **Given** ユーザーがトップページにいる、**When** 「使い方ガイド」リンクを確認する、**Then** 本のアイコン付きのボタンスタイルで表示される

---

### User Story 3 - シンプルなライトモード体験 (Priority: P3)

ユーザーはライトモード専用のクリーンなインターフェースを体験します。ダークモード切り替えボタンが表示されないため、UIがシンプルになり、混乱が減少します。

**Why this priority**: ダークモードの削除は、デザインの一貫性を保つための決定です。Skyテーマはライトモードに最適化されており、ダークモードを提供することで逆にユーザー体験が損なわれる可能性があります。

**Independent Test**: すべてのページを開き、右上にダークモード切り替えボタン（月アイコン）が表示されないことを確認できます。また、開発者ツールでコードを確認し、`dark:` クラスが含まれていないことを検証できます。

**Acceptance Scenarios**:

1. **Given** ユーザーが任意のページを開いている、**When** ページ右上を確認する、**Then** テーマ切り替えボタンが表示されない
2. **Given** ユーザーがページを読み込んでいる、**When** ページのスタイルを確認する、**Then** すべての要素がライトモードのカラー（gray-900テキスト、白背景など）で表示される
3. **Given** 開発者がコードを確認している、**When** HTML要素のクラスを検査する、**Then** `dark:` プレフィックスのクラスが存在しない

---

### Edge Cases

- **ブラウザのダークモード設定との競合**: ユーザーのOS/ブラウザがダークモードに設定されている場合でも、アプリケーションは常にライトモードで表示される
- **レスポンシブデザイン**: モバイルデバイス（320px〜768px）でボタンナビゲーションが適切に折り返し、タップしやすいサイズを維持する
- **長いテキストのナビゲーションボタン**: ボタンのテキストが長い場合でも、レイアウトが崩れない
- **複数のナビゲーションボタンの配置**: 3つ以上のボタンが横並びになる場合、画面幅に応じて適切に折り返す

## Requirements

### Functional Requirements

#### ダークモード機能の削除

- **FR-001**: システムは ThemeToggle コンポーネントを完全に削除しなければならない
- **FR-002**: システムはすべてのページから `<ThemeToggle />` の使用箇所を削除しなければならない
- **FR-003**: システムはすべてのファイルから `dark:` Tailwind CSS クラスを削除しなければならない
- **FR-004**: システムは ThemeToggle コンポーネントのテストファイルを削除しなければならない

#### Skyカラーパレットの適用

- **FR-005**: システムはすべてのページの背景に `bg-gradient-to-br from-white via-sky-50 to-white` グラデーションを適用しなければならない
- **FR-006**: システムはすべてのカード要素に `bg-white border border-sky-100 rounded-xl shadow-lg` スタイルを適用しなければならない
- **FR-007**: システムはメインボタンに `bg-gradient-to-r from-sky-500 to-sky-600 text-white hover:from-sky-600 hover:to-sky-700` スタイルを適用しなければならない
- **FR-008**: システムはすべてのアイコンに `text-sky-600` カラーを適用しなければならない

#### ボタンスタイルナビゲーションの統一

- **FR-009**: システムはすべてのナビゲーションリンクをボタンスタイル（`inline-flex items-center gap-2 px-4 py-2.5 bg-white border border-sky-200 rounded-lg text-sm font-medium text-gray-700 hover:bg-sky-50 hover:border-sky-300 hover:text-sky-700 transition-all shadow-sm`）に変更しなければならない
- **FR-010**: システムは各ナビゲーションボタンに適切なアイコン（w-5 h-5、SVG）を表示しなければならない
- **FR-011**: システムはナビゲーションボタンのホバー時にスムーズなトランジション効果を提供しなければならない

#### 対象ページ

- **FR-012**: システムは以下の6ページにデザイン変更を適用しなければならない：
  - `app/page.tsx`（組織作成ページ）
  - `app/[org]/page.tsx`（イベント一覧）
  - `app/[org]/admin/page.tsx`（管理画面トップ）
  - `app/[org]/admin/groups/page.tsx`（グループ管理）
  - `app/[org]/events/[id]/page.tsx`（イベント詳細）
  - `app/[org]/my-register/page.tsx`（一括登録）

### Key Entities

このフィーチャーは既存のデータモデルを変更せず、UIのみを更新します。

## Success Criteria

### Measurable Outcomes

- **SC-001**: すべてのページ（6ページ）で一貫したSkyカラーパレット（sky-50〜sky-600）が適用されている
- **SC-002**: すべてのナビゲーションリンクがアイコン付きボタンスタイルで表示され、ホバー時に視覚的フィードバックが提供される
- **SC-003**: ダークモード関連のコード（ThemeToggle コンポーネント、dark: クラス）が完全に削除され、コードベースに存在しない
- **SC-004**: ユーザーが任意のページを開いたとき、視覚的に統一されたデザインが即座に表示される
- **SC-005**: モバイルデバイス（320px幅）からデスクトップ（1920px幅）まで、すべての画面サイズで適切にレイアウトが表示される
- **SC-006**: すべてのページで既存の機能（イベント作成、出欠登録など）が正常に動作し、デザイン変更による機能的な影響がない

## Assumptions

- 既存のデザインシステムコンポーネント（Button, Card, Heading, Input, Message）はそのまま使用し、必要に応じてクラスを上書きする
- モックアップファイル（`samples/landing-pages/final-*.html`）が参考デザインとして利用可能
- 既存のテストは引き続き通過し、新たなテストは不要（デザインのみの変更）
- Tailwind CSS 3.4 がプロジェクトに既に導入されており、Sky カラーパレットが利用可能
